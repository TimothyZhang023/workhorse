# OpenClaw-like Web Agent Assistant 设计规范 (Spec)

## 1. 概述与背景 (Overview & Background)

随着 AI Agent 技术的不断演进，以自主决策、工具调用为核心的网页操作操作助手逐渐成为提高生产力的关键工具。本项目的主要目标是对标并超越 **OpenClaw** 的核心理念，打造一款高度自动化、具备强大的扩展能力以及用户友好的网页操作 Agent 平台。

系统旨在将零散的 AI 能力（如大模型推理、网页交互、外部 API 调用）进行深度整合，抽象出 **AgentTask**（任务实例）、**MCP 工具体系**以及 **Claude Skill** 等核心概念，并赋予其**定时调度 (Cron)** 与直观的**可视化管理界面**，形成一个功能完备的自动化执行平台。

---

## 2. 核心架构与功能模块设计

### 2.1 优化 MCP (Model Context Protocol) 协议接入支持

当前基于 MCP 的外部工具接入主要提供基础的方法调用，为适应企业级使用场景及复杂的网络安全环境，我们需要重点补齐以下能力：

#### 2.1.1 动态 HTTP Header 支持

- **需求说明**：在连接至远端 MCP Server 或直接调用 REST API 时，许多系统需要特定的请求头来进行路由校验、灰度控制或环境标识。
- **功能实现**：
  - 在注册 MCP Tool / Server 的配置页面中，增设 `Headers` 配置项（Key-Value 对形式）。
  - 支持变量解析机制（例如：`tenant_id: {{user.tenant_id}}`，`X-Correlator: {{uuid()}}`），在 Agent 执行时动态渲染。

#### 2.1.2 认证与鉴权 (Authentication & Credentials)

- **需求说明**：多数企业内部服务或经过身份验证的 MCP Server 均要求提供凭证信息。
- **功能实现**：
  - **静态认证**：支持常见的 `Basic Auth` 账号密码、`API Keys` 和长效 `Bearer Token` 的配置并自动随请求注入。
  - **安全存储**：用户的 Auth 凭据必须在服务端使用加密算法存储，确保在前端界面获取配置和执行日志时均为脱敏（Masking）状态。
  - 全局 Auth 配置可与特定的 MCP Server 绑定。

### 2.2 强化 Claude Skill 技能体系框架支持

单纯的工具提供（Tools）往往过于底层，要求大模型从零开始推理每个步骤。为了提升 Agent 完成特定领域任务的效率与准确率，系统将引入 **Claude Skill** 的概念。

#### 2.2.1 Skill 的定义结构

Skill 相当于一个"能力包"或者"微型智能体原型"，它包含：

1.  **System Prompt 覆盖/增强**：提供专门针对该领域的系统人设和思考逻辑指导（例如：如何正确地使用 XPath 定位器来提取电商商品价格）。
2.  **特定的 Few-Shot 示例集**：提供过去类似任务成功执行的历史记录（Thought/Action 对），供模型进行上下文学习。
3.  **约束与工具白名单**：该技能在被激活时推荐或限定必须搭载的一些 MCP 工具。

#### 2.2.2 运作模式

- 用户可以在**技能库**中浏览、复刻、或者从头创建一个 Skill（如：“网页内容抓取及去重归纳技能”、"日常发票报销审批技能"）。
- 在建立新的 AgentTask 时，系统支持通过复选框将一个或多个 Skills 组合注入到当前 AgentTask 的上下文中。

---

### 2.3 `AgentTask` 任务定义架构 (The ReAct Loop Engine)

`AgentTask` 是此系统最具代表性的执行单元载体，它本质上是用户为了完成某一个/一类具体工作所创建的配置集合与隔离运行状态。

#### 2.3.1 AgentTask 的核心组件

- **System Prompt**：定义这次任务的具体目标、约束和执行上下文。
- **ReAct Loop 调度器 (推理-执行循环)**：
  - `Thought (思考)`: Agent 获取到当前网页截图或 DOM 树后的思考过程。
  - `Action (行动/工具调用)`: 决定下一步要调用的 MCP 能力（诸如“点击按钮”、“输入文本”、“调用第三方天气 API”）。
  - `Observation (观察)`: 执行完成系统回传的反馈结果。
  - 循环往复直至模型推断目标已达成并发出 `Finish` 或 `Stop` 指令。
- **Tools (MCP) & Skills 挂载点**：界定这个单体 Agent 所能获得的“手”和“脑”。

#### 2.3.2 运行隔音与状态留存

每个 AgentTask 运行时将会在后台或者独立的浏览器上下文环境 (Browser Context) 中跑，避免相互干扰。同时记录整个 ReAct Loop 的调用树（Trace），便于进行日志分析与 Prompt 调优。

---

### 2.4 Cron 定时任务与自动化调度

Agent 的最终归宿是作为"无人值守"的助手来提高生产力。增加基于时间维度的调度系统是闭环的关键。

#### 2.4.1 任务调度类型

- **一次性延迟执行**：例如 “在今天晚上 11 点执行”。
- **周期性定时执行 (Cron Jobs)**：完全支持 Unix 标准的 Cron 表达式（如 `0 0 * * *` 每日零点执行）。

#### 2.4.2 调度管理系统设计

- 在前端配置时，用户应可以为一个指定的 `AgentTask` 附加一个调度策略。
- **异常与重试机制**：由于网页执行的不确定性，需要支持失败后一定次数内的自愈重试策略（如间隔 5 分钟重试 3 次）。
- **执行历史与看板**：直观展示哪些定时任务成功，哪些超时或出错，并在失败时支持发出一键重试指令。

---

### 2.5 导航栏及前端 UI 整体整合与交互

为了让用户能有一个一站式的工作体验，需将上述概念转化为可视化的产品形态，并整合到平台左侧的主导航栏中。

#### 导航栏 (Siderbar) 结构设计建议：

```text
[系统 Logo] WebAgent Platform
---------------------------------------
📊 探索 & 仪表盘 (Dashboard)             --> 总体运行概览、执行成功率核心指标

🤖 我的 Agent 助手 (My Agents)
    ├── 📝 任务编排 (Agent Tasks)         --> 核心界面：创建 Task，配置 Prompt、挑选 Skills 与 Tools。启动和单步调试。
    └── ⏱️ 定时调度 (Cron Jobs)           --> 为 Task 附加时间计划，包含日历视图、Cron 表达式可视化配置及调度监控。

⚙️ 资源与能力 (Resources)
    ├── 🛠️ MCP 接入配置 (MCP Servers)     --> 注册远端/本地 MCP Server、配置 HTTP Header、Auth 认证信息及管理凭证安全。
    └── 🧠 技能库 (Skills Library)         --> 管理、导入、导出 Claude Skills，编辑预设的 prompt 和 few-shot。
```

#### 用户使用路径 (User Journey)

1. **准备阶段**：在 `MCP 接入配置` 与 `技能库` 准备好“弹药”（配置好所需的数据接口服务和特定场景技能）。
2. **编排阶段**：在 `任务编排` 组装你的 AgentTask，通过左侧拖拽选择刚才的 Skill 及 MCP，编写 Prompt 后可以在线 `Run & Debug`。
3. **闭环阶段**：测试没问题后进入 `定时调度`，挂上每天执行的闹钟，Agent 即可进入无人值守运转状态。
